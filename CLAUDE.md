# Instrucution

## Rules to follow:
1. Always check your source, implementaion, and everything in plan with the latest documentation and codebase of the project. you can check with context7 and internet for up-to-date stable information.
2. You can not add, update or remove directly with migration files. purpose of migration file is to keep track of changes in the database schema.

## Definition Of Done
- Code is clean, well-structured, and follows best practices.
- All tests are passing, and new tests have been added for any new functionality.
